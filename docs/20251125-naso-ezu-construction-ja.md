# 「な..そ」「え..ず」構文におけるgloss記述原則

Last updated: 2025/11/25-15:54:28

本ドキュメントは、古典日本語に見られる呼応構文「な..そ」「え..ず」などの記述において、gloss（語機能注）の原則と具体例を整理するものである。語の表面的な形態や伝統的な品詞分類ではなく、意味素性（semantic feature）と構文的役割に基づいて記述することを目的とする。

---

## 1. 「え..ず」構文（不可能）

### 例：

```json
[
  { "word": "え", "gloss": "POT^1" },
  { "word": "まさら", "gloss": "surpass.IRR" },
  { "word": "ず", "gloss": "NEG^2" }
]
```

### 注記：

> **POT^1 + NEG^2**：「え..ず」は古典日本語における不可能表現の構文である。「え」は可能性を示す前項成分であり、「ず」によって否定されることで構文全体として不可能を表す。

---

## 2. 「な..そ」構文（禁止）

### 例：

```json
[
  { "word": "な", "gloss": "PROH^1" },
  { "word": "泣き", "gloss": "cry.ADV" },
  { "word": "そ", "gloss": "CMD^2" }
]
```

### 注記：

> **PROH^1 + CMD^2**：「な…そ」は古語に見られる柔らかい禁止表現である。「な」が禁止構文の導入部として機能し、「そ」（「す」の命令形に由来する可能性がある）が終止要素として構文を閉じる。

---

## 3. gloss記述の原則

- glossは分類名（例：「終助詞」）ではなく、**語の機能や意味素性を表すこと**
- 呼応構文で語が離れている場合のみ `^1`, `^2` を用いて **構文要素の開始と終了を示す**
- 「にけり」「てむ」などのように語が連続し、意味的に一体化している場合は、**一語の gloss として処理し、構文項目として扱わない**
- 語の歴史的由来は注記で補足するが、gloss自体は機能を優先

---

## 4. 今後の追加予定

- 呼応構文の変遷と位置移動（例：「な」や「ぞ」の文中→文末化）

※ 「にけり」「てむ」のように語が連続し意味が一体化しているものは、構文としてではなく一語の gloss として処理すれば十分であるため、独立した構文項目は不要とする。

これらも同じ原則に従って記述する：語ごとに最小限の gloss を付し、構文上の機能を明示し、必要に応じて歴史的または機能的な注記を加える。
