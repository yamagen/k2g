# K2G + FPS-ID + CW パイプラインの実現可能性について

## 【はじめに】

本ドキュメントでは、古語解析パイプライン「K2G（形態素解析）+ FPS-ID（意味ID生成）+ CW（共起重み計算）」の**技術的・理論的な実現可能性**について、設計・実装・評価の3段階に分けて明確に論じる。

---

## 【1. 設計段階の実現性】

### 1.1 データ資源の充実

- AEAD（An Expression A Day）、『伊勢物語』『土佐日記』などにより、すでに十分な量の **gloss・逐語訳・phrase-gloss 構造**が蓄積されている

### 1.2 アーキテクチャの独立性

- fastText や BERT などの大規模モデルには依存せず、**頻度・語順・共起**に基づく軽量な構造で設計されている

### 1.3 分割・辞書仕様の整備

- Leipzig Glossing Rules 準拠で語順・意味機能が明示されており、辞書への機械的変換が可能（gloss = 機能タグ）

### 1.4 実装対象の明確化

- すべてのモジュール（k2g, fps-id, cw）は **C言語**により実装可能であり、UNIXツールチェインと連携できる構造を採用

---

## 【2. 実装段階の実現性】

### 2.1 `k2g-analyze`

- glossデータに基づく語分割・品詞付与
- 接続規則を用いず、最大一致と辞書照合により構成（既存のAEAD構造と整合）

### 2.2 `fpsid-gen`

- 共起カウントは文脈スライドウィンドウ方式（±n）で処理可能
- PCA計算は LAPACK等のCライブラリで対応可能
- 出力は "23.6147" のような小数表現で意味階層を含むIDとして整理

### 2.3 `cw`

- `fps-id`の先頭桁による語義クラスタ制御（digit指定）
- 重み式：`cw(t1, t2) = ctf × √(idf1 × idf2)` はCで直接実装可能
- 出力は `.dot` 形式で Graphviz へ受け渡し可

### 2.4 可視化（Graphviz）

- 語義ネットワークは `.dot` → `.svg` により視覚出力可能（語義距離・重みをノード・エッジに反映）

---

## 【3. 評価段階の実現性】

### 3.1 出力の可読性

- K2G の出力は JSON形式で、人手訳や gloss と比較しやすい
- FPS-ID は階層的意味識別子として、文脈ごとの語の変化を示せる

### 3.2 意味グラフの解釈性

- 意味クラスタ構造は、文脈的に一貫した語群を可視化できる（例：「春」「花」「咲く」などが同クラスタ）

### 3.3 既存システムとの比較

- MeCab では「いますがり」「かくなりぬ」などの構造を正しく解析できないが、K2G では gloss により可能
- BGH（分類語彙表）や word2vec の類似語は階層性・意味解像度制御に乏しいが、FPS-IDはそれを補完

### 3.4 評価方法の多様性

- 意味グラフの納得性・訳語照合・類義分布の妥当性など、**構造・統計・直感**の三面から評価が可能

---

## 【結論：なぜ実現可能か】

| 項目      | 実現可能性の根拠                                  |
| --------- | ------------------------------------------------- |
| ✅ データ | AEAD / gloss / 古典語訳語コーパスがある           |
| ✅ 技術   | C言語 + UNIX による軽量設計、依存少なし           |
| ✅ 評価性 | 出力が視覚・構造・数値で検証可能                  |
| ✅ 方法論 | 統一構造（辞書 → ID → 重み → グラフ）で設計が一貫 |

このK2G + FPS-ID + CWパイプラインは、**詩学・語用論・構文研究・教育**のいずれにおいても、
「古語テキストの意味構造を解析し、かつ可視化・操作可能にする」という目的に対し、
**段階的かつ確実に実現できる研究基盤である**。
