### 📋 **gloss 記述ガイドライン（簡易版）**

---

#### ✅ **1. 基本原則**

- gloss は形態・意味を表すラベルを付け、文法要素を精密に記述する。
- gloss 記号は **nonLGR-abbreviations-ja.json / nonLGR-abbreviations-en.json** に定義されたものを用いる。
- LGR（Leipzig Glossing Rules）準拠ラベルと、プロジェクト独自ラベルの両方を採用。

---

#### ✅ **2. ラベルの範囲**

| 分類     | 説明                                                                                           | 例                                                  |
| -------- | ---------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| 品詞     | 動詞（V）、名詞（N）、形容詞（ADJ）、助動詞（AUX）など                                         | `do`（する）、`see`（見る）、`AUX`（助動詞）        |
| 形態     | 活用形や接続形態を表す。`NEG.BASE`（未然形）、`ADV.F`（連用形）、`LEX.F`（終止形）など         | `ikazu`（行かず）、`ikite`（行きて）、`iku`（行く） |
| 意味機能 | 意味的役割を表す。`CNTFCT`（反実仮想）、`POT`（可能）、`HESIT`（ためらい）、`WISH`（願望）など | `まし`（CNTFCT）、`られる`（POT）                   |

---

#### ✅ **3. ラベル付けの方針**

- **意味が一意に決まる場合** → 意味ラベルのみ（例：`CNTFCT`）。
- **意味ラベルと形態を区別したい場合** → 意味＋形態（例：`CNTFCT.AUX`、`POT.AUX`）。
- **終止形（finite form）は通常省略可** → 必要なら `.LEX.F` を付記。

---

#### ✅ **4. 記述例**

| 要素               | gloss 記述                        |
| ------------------ | --------------------------------- |
| 「わかれせましや」 | `part.NEG.BASE do.ADV.F CNTFCT Q` |
| 「見ましかば」     | `see.ADV.F CNTFCT COND`           |
| 「行かず」         | `go.NEG.BASE NEG`                 |
| 「食べられる」     | `eat.ADV.F POT`                   |
| 「行ける」         | `go.ADV.F POT`                    |

---

#### ✅ **5. gloss 定義ファイル**

- 日本語版 → `nonLGR-abbreviations-ja.json`
- 英語版 → `nonLGR-abbreviations-en.json`

⚠ 定義ファイルに登録されていない新ラベルを使う場合は、必ず事前に gloss 定義を追記してから使用する。

---

#### ✅ **6. 管理と整備**

- gloss ラベルは短縮形（例：`CNTFCT`、`POT`）を使う。
- 定義ファイル内に意味・例を明記し、外部共有時の用語説明とする。
- 形態ラベル（例：`.ADV.F`、`.NEG.BASE`）は統一し、語ごとに付与基準を揃える。
